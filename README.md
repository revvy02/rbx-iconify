# rbx-iconify

Use any of the 200,000+ open source icons on [Iconify](https://icon-sets.iconify.design/) in your Roblox game by referencing them by name in your Luau code.

`rbx-iconify` scans your Luau sources for icon references like `"lucide:x"`, fetches the icons from the Iconify API, and generates:

- **Sprite sheet PNGs**: white-on-transparent tiles, tintable at runtime via `ImageColor3`
- **A Luau index module**: maps each icon name to its sheet, offset, and size

## Installation
Install via `mise` or `rokit`

```sh
# mise
mise use ubi:revvy02/rbx-iconify

# rokit
rokit add revvy02/rbx-iconify
```

## Quick start

Create an `iconify.toml` in your project root:

```toml
scan = ["src/**/*.luau"]
default_set = "lucide"

[output]
index = "src/shared/iconify.luau"
sheets = "assets/images/iconify"

[target.ui]
tile_size = 64
columns = 16
```

Reference icons anywhere in your scanned sources as string literals:

```lua
someIcon("lucide:x")
someIcon("ph:gear-six")
```

Run `rbx-iconify` from the project root (or `rbx-iconify --watch` during development). Each target gets its own directory under `output.sheets`, with numbered sheets inside it (here `assets/images/iconify/ui/1.png`), and the index module is written at `output.index`.

The generated module is a lookup function:

```lua
local iconify = require(path.to.iconify)

local icon = iconify("lucide:x", "ui")
```

Upload the sheet PNGs as image assets. An asset tool like [Asphalt](https://github.com/jacktabscode/asphalt) automates the upload and generates the name → asset id map for you (the examples below assume `style = "nested"` and `strip_extensions = true` in Asphalt's codegen config).

Typical usage is a small `Icon` component that wraps the lookup. With [react-lua](https://github.com/jsdotlua/react-lua):

```lua
local iconify = require(path.to.iconify)
local images = require(path.to.images).iconify -- @generated by Asphalt

local function Icon(props)
	local target = props.target or "ui"
	local data = iconify(props.name, target)

	return React.createElement("ImageLabel", {
		Image = images[target][tostring(data.sheet)],
		ImageRectOffset = data.offset,
		ImageRectSize = data.size,
		ImageColor3 = props.color, -- icons are white, tint freely
		Size = props.size or UDim2.fromOffset(data.size.X, data.size.Y),
		BackgroundTransparency = 1,
	})
end

-- React.createElement(Icon, { name = "lucide:x", color = Color3.fromRGB(255, 170, 0) })
```

Or via Miu (immediate mode UI)

```lua
local function Icon(props, renderChildren)
	local target = props.target or "ui"
	local data = iconify(props.name, target)

	return Instance("ImageLabel", {
		Image = images[target][tostring(data.sheet)],
		ImageRectOffset = data.offset,
		ImageRectSize = data.size,
		ImageColor3 = props.color or Color3.new(1, 1, 1),
		Size = props.size or UDim2.fromOffset(data.size.X, data.size.Y),
		BackgroundTransparency = 1,
	}, renderChildren)
end

-- Icon({ name = "lucide:x", color = Color3.fromRGB(255, 170, 0) })
```

Sheets are capped at 4096px; overflow spills into `2.png`, `3.png`, etc. `icon.sheet` tells you which, so keying the asset map by sheet number keeps overflow transparent to call sites. Each target's sheet directory is owned by the generator: stale numbered sheets from a previously larger icon set are deleted on regeneration (though renaming or removing a target leaves its old directory behind).

## Watch mode

`rbx-iconify --watch` re-runs generation whenever scanned `.luau` sources change. Runs are debounced, regeneration is skipped when the set of icon references is unchanged, and the generated index module is ignored so the tool doesn't retrigger on its own output. `iconify.toml` is read once at startup, so config changes require a restart.

## Generic names

The `default` pseudo-set decouples call sites from a specific icon set:

```toml
default_set = "lucide"

[default]
close = "lucide:x"
settings = "ph:gear-six"
```

```lua
someIcon("default:close")        -- → lucide:x (explicit alias)
someIcon("default:chevron-down") -- → lucide:chevron-down (falls back to default_set)
```

Swap the whole icon set for your game by changing one line of config.

## Style overrides

Styles cascade in three layers, from set defaults to per-icon overrides to call-site overrides:

```toml
[lucide]
stroke_width = 2       # applies to every lucide icon

["lucide:x"]
stroke_width = 3       # just this icon
```

```lua
someIcon("lucide:x:stroke_width=1") -- call site wins; rendered as its own variant
```

Each distinct override combination becomes its own tile on the sheet. Currently `stroke_width` is the supported style property.
